IMapMatchingParameters
Summary
Provides access to the MapMatching parameters which control the operation of the map matching COM function.
Description

The attribute TSysSet controls which transport systems shall be considered or if transport systems shall be considered at all.

If TSysSet is empty (as default), all links and turns are considered in the map matching procedure, regardless of their actual TSysSet setting. If you want to restrict the map matching to a set of specific transport systems while still taking closed turns into account, you need to set TSysSet to the according concatenated transport system codes and set RouteOverClosedTurns=False

Please Note: The 'Incremental' matching algorithm has been removed in PTV Visum 18. Only the 'Global' algorithm is available and the attribute 'Algorithm' for selecting the matching algorith is therefore obsolete.
