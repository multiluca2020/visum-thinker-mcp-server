IFilters
Summary
The object allows you to access filter settings
Object Model
Description
The object allows you to access filter settings. Filter settings are read from or written to a file, or defined for each individual filter via the COM interface. For most network objects the filter settings are made via an ISingleFilter object. The access function used to create the filter object defines what type of filter it is. Besides the ISingleFilter, there are other filter objects with special properties for hierarchical filters.
To define a filter, add filter conditions that are specified using range objects. A range object describes the permissible value range "from X to Y" (including), "greater than or equal to X" or "smaller than or equal to Y". Bracketed conditions are defined using a helper object which is provided by IFilters itself.
Examples
-
Define link filter (IDirectedFilter) Example code snippet (VBA):Define link filter (IDirectedFilter)
  • Visual Basic
  • Python 
Set LinkFilter = Visum.Filters.LinkFilter
LinkFilter.Undirected = False
LinkFilter.UseFilter = True
LinkFilter.AddCondition "OP_NONE", False, "NO", "ContainedIn", Visum.Filters.Range(100, 10000)
Visum.Filters.LinkFilter().Init()
#Condition to filter transport systems 'C' and 'H'
cond1 = Visum.Filters.LinkFilter().AddCondition("OP_NONE",False,"TSYSSET","ContainsOneOf","C,H")
#Condition to filter links with capacity between 700 and 1200
rng1 = Visum.Filters.Range(700,1200)
cond2 = Visum.Filters.LinkFilter().AddCondition("OP_AND",False,"CAPPRT","ContainedIn",rng1)
#Condition to filter links where volume exceeds capacity, insert after condition 1
cond3 = Visum.Filters.LinkFilter().AddCondition("OP_AND",False,"VOLVEHPRT(AP)","GreaterAtt","CAPPRT",1)
#Condition to filter links where reverse direction is blocked
cond4 = Visum.Filters.LinkFilter().AddRelationCondition("OP_OR",False,"ReverseLink","ALL","TSYSSET","EqualVal","")
#put first 3 conditions in brackets
Visum.Filters.LinkFilter().AddBrackets(1,3)
