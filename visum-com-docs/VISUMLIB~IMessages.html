IMessages
Summary
Access to all messages generated during COM execution
Description
During execution of a COM script all error and warning messages which would normally be displayed in a message box are suppressed so that execution of the script is not blocked. In many cases it may be useful to analyze these messages within the COM script, so that the script can react to exceptional situations. Complex operations like reading a network file can produce numerous messages, and the status of the operation (success or failure) depends on the contents of the messages.
From Visum 9.2 onwards, all error and warning messages are accumulated in a special collection object, accessed from the Visum object. The script is responsible for checking whether the collection is non-empty and will then iterate over the messages and interpret them. When the script has processed all accumulated error messages, it clears the contents of the collection.
