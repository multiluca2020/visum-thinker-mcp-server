EstimateTypicalCostPer1000Meters Method
Summary

Syntax
Visual Basic
Public Sub EstimateTypicalCostPer1000Meters()
Description

The 'Global' map matching algorithm will consider a path between two input points as a detour, if the cost on this path are greater than the straight distance between the input points and on that base choose between path options. If the cost attribute used for map matching is in different units or not representing a distance at all (e.g. travel time), the attribute TypicalCostPer1000Meters can be used to provide a conversion factor between the two measures.

Use this function to calculate a value for this attribute before the actual mapping is done. This is only required, if the attributes LinkCostAttr and TurnRelCostAttr point to attributes which do not represent distances in meters.



Examples

-
  • Python 
mm = Visum.Net.CreateMapMatcher()                               # create a new MapMatcher
mm.MapMatchingParameters.SetAttValue("TSYSSET", "CAR,BIKE")     # limit to TSys 'CAR' and 'BIKE'
mm.MapMatchingParameters.SetAttValue("ROUTEOVERCLOSEDTURNS", 1) # don't route over closed turns
mm.MapMatchingParameters.SetAttValue("MATCHRADIUS", 100)        # match to links max. 100m
mm.MapMatchingParameters.SetAttValue("ALGORITHM", 1)            # use 'global' algorithm
mm.MapMatchingParameters.SetAttValue("STANDARDDEVIATION", 70)   # Set the weighting between match distance and route length

mm.MapMatchingParameters.SetAttValue("LinkCostAttr", "t0_PrTSys(CAR)") #use car travel time for routing costs on links
mm.MapMatchingParameters.SetAttValue("TurnRelCostAttr", "t0_PrTSys(CAR)") #use car travel time for routing costs on turns
mm.MapMatchingParameters.EstimateTypicalCostPer1000Meters()     # estimate the conversion factor between travel time and distance

mm.MatchPointSequence(routePointCoords)                         # do the matching for all points at once
