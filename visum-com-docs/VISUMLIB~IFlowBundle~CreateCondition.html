Summary
Creates an object flow bundle condition for a network object with restricted activity type.
Syntax
Visual Basic
Public Sub CreateCondition( _
   ByVal NetElm As Variant, _
   ByVal ActivityTypeSet As IActivityTypeSet, _
   Optional ByVal Complement As Boolean = False, _
   Optional ByVal ConsiderTimeRestriction As Boolean = False, _
   Optional ByVal FromTime As Integer = 0, _
   Optional ByVal ToTime As Integer = 86400 _
)
Parameters
NameValue typeDescription
NetElmVariantA network element for object flow bundle calculation
ActivityTypeSetIActivityTypeSetIActivityTypeSet object that describes the activity examined
ComplementBooleanSet to True, to use the complement of the defined condition
ConsiderTimeRestrictionBooleanApply temporal restrictions referring to a simulation-based assignment
FromTimeIntegerStart time of the temporal restriction
ToTimeIntegerEnd time of the temporal restriction
Description
The condition is added to the last possible term, i.e. it is linked with the previous condition using an "AND" operation (if the last term was not closed by calling the CreateNewGroup or an Execute… method).
Examples
-
  • Python 
myFlowBundle = Visum.Net.DemandSegments.ItemByKey("P").FlowBundle # Choose your PuT Demand segment
myFlowBundle.Clear() # Initialize myFlowBundle
stoppoint20 = Visum.Net.StopPoints.ItemByKey(20)
myActivityTypeSet = myFlowBundle.CreateActivityTypeSet()
myActivityTypeSet.Add(124) # All kinds of PuT Passenger types: 4 + 8 + 16 + 32 + 64 = 124
myFlowBundle.CreateCondition(stoppoint20, myActivityTypeSet)
