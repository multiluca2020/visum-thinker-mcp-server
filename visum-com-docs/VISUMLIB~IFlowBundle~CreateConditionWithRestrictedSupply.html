Description
Additionally, you can specify offer limitations using an INetElements container: For each network object type (operator, main line, line route, time profile, vehicle journey and transport system) for which there is an object in the container, you have to use the objects specified. If there is no object in the container for one of the types, then there is no such limitation. In addition to the INetElements object, the localSupply_includeWalk parameter is an option that allows you to add PuT pedestrian transport systems to the offer.
The second set of analogous offer parameters is only interpreted for stop (area, point) object flow bundles and describe a more detailed offer for passengers in transit. If this container is not specified, there are no offer limitations for a more detailed offer.
If a group starts with a zone or main zone condition, but only destination traffic is required, an error message is displayed. This is also the case, if a group ends with a zone or main zone condition and only origin traffic is requested.
