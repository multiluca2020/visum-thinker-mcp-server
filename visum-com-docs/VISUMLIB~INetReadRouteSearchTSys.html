INetReadRouteSearchTSys
Summary
Parameter object to define parameters for automatic line route completion
Description

INetReadRouteSearch defines the options for the line and system route paths completion during the net reading process. It is also used to control the routing of new line routes created through the methods AddLineRoute and AddSystemRoute.
It is created through the CreateNetReadRouteSearchTSys Method.

The interface INetReadRouteSearchTSys is to be filled with the data from the former interface (method SearchShortestPath() at INetReadRouteSearchTSys) as follows:

former property actual attribute
shortestPathCriterion ShortestPathCriterion
includeBlockedLinksInRouting IncludeBlockedLinks
includeBlockedTurnsInRouting IncludeBlockedTurns
maxDeviationFactor MaxDeviationFactor
doIfNotFound WhatToDoIfShortestPathNotFound
linkTypeIfInsert SetAttValue(“LinkTypeForInsertedLinksReplacingMissingShortestPaths”, linkTypeIfInsert.AttValue(„No“))
doIfStopPointNotFound  WhatToDoIfStopPointNotFound
doIfStopPointIsBlocked WhatToDoIfStopPointIsBlocked
SetLinkTypeIfOpened ChangeLinkTypeOfOpenedLinks
LinkTypeIfOpened LinkTypeForOpenedLinks

To meet the former behavior the attribute HowToHandleIncompleteRoute must be set to RouteSearchHandleIncompleteRouteTSearchShortestPath additionally.

The method InsertOrOpenLink was replaced by the attribute LinkTypeForInsertedLinksReplacingMissingLinks
