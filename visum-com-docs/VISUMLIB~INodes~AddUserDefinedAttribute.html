Summary
Adds a user-defined attribute to this net object type

Currently, only time interval sets can be specified as subattribute dimensions. The are refered to by a string 'TISET_' + number of the time interval set. When only the standard analysis times are present in the network (in older models), they are thus usually referred to as 'TISET_1'. Ommit the argument for creating UDAs without the subattribute dimension.

CanBeEmptyVariant
Description

The ID must not already be used by a predefined or already existing user-defined attribute of this net object, or an error is thrown.
Note that in case of length attributes (long length or short length), the value of minval, maxval and defval parameters has to be specified using unit [m] even for long length (unit [km / mi]) and for networks using imperial units.
 

Examples
-
  • Python 
.. .AddUserDefinedAttribute("UDA_INTEGER","UDA_INTEGER","UDA_INTEGER",1) #creates a user-defined attribute of type 'integer'
.. .AddUserDefinedAttribute("UDA_FLOAT","UDA_FLOAT","UDA_FLOAT",2) #creates a user-defined attribute of type 'float'
.. .AddUserDefinedAttribute("UDA_BOOL","UDA_BOOL","UDA_BOOL",9,defval=True) #creates a user-defined attribute of type 'bool' with default value of 'Yes' using named arguments
.. .AddUserDefinedAttribute("UDA_TEXT","UDA_TEXT","UDA_TEXT",5)  #creates a user-defined attribute of type 'text'
.. .AddUserDefinedAttribute(ID="UDA_FORM1",ShortName="UDA_FORM1",LongName = "UDA_FORM1",VT= 2,Formula = "[ADDVAL1] + [AddVal2]") #creates a user-defined attribute using a formula. This special treatment with named parameters is due to a Python bug.
.. .AddUserDefinedAttribute("UDA_FORM2","UDA_FORM2","UDA_FORM2",2,0,0,0,0,0,0,0,"[ADDVAL1] + [AddVal2]")  #use this variant if the one using named arguments shown above does not work
.. .AddUserDefinedAttribute("UDA_INTEGER1","UDA_INTEGER1","UDA_INTEGER1",vt = 1,subAttr = 1 ) #creates a user-defined attribute of type 'integer' with subattribute AHPI
