Summary
This method opens Visum’s attribute selection dialog associated to the net object type.
Syntax
Visual Basic
Public Function AskAttribute( _
   Optional ByVal EditableOnly As Boolean = False, _
   Optional ByVal NumericOnly As Boolean = False, _
   Optional ByVal AllowMultiSelection As Boolean = False, _
   Optional ByVal DefaultID As String = "", _
   Optional ByVal Mask As String = "", _
   Optional ByVal AttrCategories As String = "", _
   Optional ByVal SpecialFirstEntry As String = "" _
) As String
Parameters
NameValue typeDescription
EditableOnlyBooleanIf true, the dialog only shows the editable attributes
NumericOnlyBooleanIf true, the dialog only shows the numerical attributes
AllowMultiSelectionBooleanIf true, the dialog allows multi selection of attributes
DefaultIDStringIf not empty, an attribute ID of a net object type or a value taken from specialFirstEntry. This entry will be selected initially.
MaskStringA string containing wildcards (? and *). Only direct attributes whose ID matches the mask will be provided. Indirect attributes cannot be presented in this case.
AttrCategoriesStringA comma separated string with categories. The dialog shows only attributes of this categories.
SpecialFirstEntryStringAdditional attribute IDs for the attribute tree. Multiple entry can be represented by one string using the pipe character “|” as delimiter.
Return Type
  • ID of the selected attribute
  • Empty, if dialog was closed by 'OK' without atttribute selection.
  • defaultID, if dialog was closed by 'Cancel'
Description

The set of attributes offered in the dialog is controlled by the parameters. The method returns a string value. The string is empty if the user quits the dialog by hitting the OK button without selecting an attribute. If the user quits the dialog by hitting OK when an attribute name is selected or by double clicking an attribute, the string will be the ID of the selected attribute or the text specified by specialFirstEntry. If the user cancels the dialog, the string will be the defaultID.
Additional attribute IDs can be added to the attribute tree using the specialFirstEntry parameter.
