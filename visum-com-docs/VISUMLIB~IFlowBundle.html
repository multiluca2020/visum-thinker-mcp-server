IFlowBundle
Summary
Represents the flow bundle and controls flow bundle calculation.
Object Model
Description
The IFlowBundle interface allows you to calculate and save object flow bundles. You can calculate object flow bundles for PrT and for PuT. The object may be accessed from the INet object or from the IDemandSegment object.
Conditions are defined with respect to a demand segment set. To do so, the IFlowBundle object carries a demand segment set information. Conditions inherit this demand segment set that is valid at the moment when the condition is defined. Since any condition any only refer either to PrT or t PuT demand segments, the IFlowBundle’s current demand segment may contain either PrT or PuT demand segments only.
If the IFlowBundle object is accessed via the IDemandSegment object, the current demand segment set is set to this parent demand segment only; nevertheless it may be modified. When accessing through the INet object, the current demand segment set is empty and must be modified before defining the first condition.
First conditions are defined for the object flow bundle. Each condition corresponds to a row in the dialog. As within the dialog, the conditions within one group are combined using “and then” logic. This means this is not a logical “AND” conjunction since the order of conditions plays a role: A path must meet the conditions in the same order as they are defined.
By calling the CreateNewGroup method or one of the Execute… methods, you can specify several conditions linked with an "OR" operation. Once a condition has been defined, call Execute to execute the object flow bundle and call Save to save the results matrix.
IFlowBundle is not a net object in the narrow sense, i.e. the standard properties AttValue and Active are not provided.

fb.Execute(NetElementContainer)
